can orient quickly without reading every file.

## Architecture

**Authoritative source:** `docs/spec/frozen/` — the frozen constitutional spec set.

These files are imported and locked. They define the architectural model,
ownership rules, field encodings, and bring-up ladder. They are not edited
during implementation; they are the reference against which implementation
is validated.

Key files:
- `master_design_statement.md` — top-level design intent and philosophy
- `appendix_a_field_dictionary.md` — all shared field encodings (ENTRY_*, CM_*, FC_*, etc.)
- `appendix_b_ownership_matrix.md` — what each module is allowed to own
- `appendix_d_bringup_ladder.md` — rung definitions and gate criteria

## Shared RTL Constants

**Authoritative source:** `rtl/include/keystone86_pkg.sv`

This is the single source of truth for all shared RTL constants:
ENTRY_*, SVC_*, SR_*, FC_*, CM_*, STAGE_*, REG_*, C_*, MSEQ_* state codes.

All RTL modules must use:

```systemverilog
import keystone86_pkg::*;
```

**The legacy `.svh` files** (`entry_ids.svh`, `fault_defs.svh`, `commit_defs.svh`,
`field_defs.svh`, `service_ids.svh`) contain the same constants as backtick macros.
They exist for compatibility with external tooling only. RTL source files
must not use `\`include` for these headers — use the package import instead.

`keystone86_pkg.sv` is generated from `tools/spec_codegen/appendix_a_codegen.json`.
Do not edit it manually. To regenerate: `make codegen`.

## Generated Bootstrap Artifacts

**Source:** `scripts/ucode_build.py`
**Generated outputs:** `build/microcode/ucode.hex`, `build/microcode/dispatch.hex`, `build/microcode/ucode.lst`

These are bootstrap-phase generated files. They are regenerated by `make ucode`.
Do not edit them manually — edit `ucode_build.py` and re-run `make ucode`.

All generated build artifacts go under `build/`. The full build directory layout is:

```
build/microcode/     ← ucode.hex, dispatch.hex, ucode.lst, dispatch.lst
build/sim/rung0/     ← rung 0 .vvp and intermediates
build/sim/rung1/     ← rung 1 .vvp and intermediates
build/sim/rung2/     ← rung 2 .vvp and intermediates
build/sim/rung3/     ← rung 3 .vvp and intermediates
build/sim/rung4/     ← rung 4 .vvp and intermediates
build/sim/rung5_pass2/ ← Rung 5 Pass 2 .vvp and intermediates
build/sim/rung5_pass3/ ← Rung 5 Pass 3 .vvp and intermediates
build/sim/rung5_pass4/ ← Rung 5 Pass 4 .vvp and intermediates
build/sim/rung5_pass5/ ← Rung 5 Pass 5 .vvp and intermediates
build/synth/         ← ECP5 synthesis outputs (future)
build/formal/        ← SymbiYosys outputs (future)
```

`build/` is gitignored. Run `make clean` to remove all generated artifacts.

The Python constants in `ucode_build.py` (CM_EIP, CM_JMP, etc.) are locally
defined for bootstrap purposes. They replicate the Appendix A encodings
intentionally — the Python layer has no import mechanism to pull from
`keystone86_pkg.sv`. This is a known bootstrap limitation, not a bug.

## Microcode Source

**Authoritative source for microcode organization and review:** `microcode/src/` (`.uasm` files)

Bootstrap entries are in `microcode/src/entries/`.
The shared fault handler is in `microcode/src/shared/`.

The current effective bootstrap ROM generator remains `scripts/ucode_build.py`;
the build script embeds the bootstrap ROM content directly rather than parsing
`.uasm` files. The `.uasm` files remain useful for source organization and
review, and they are the intended future canonical source for microcode content.
No microcode-generation refactor is part of the current Rung 5 documentation
catch-up.
