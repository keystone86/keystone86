# Reviewer Changes Log

## microcode/build/dispatch.hex

**Why changed:** Critical correctness error. Entries for ENTRY_PREFIX_ONLY
(0x12) and ENTRY_NOP_XCHG_AX (0x13) had their dispatch uPC addresses
swapped. ENTRY_RESET (0xFF) pointed to ENTRY_NULL's address (0x010)
instead of its own bootstrap address (0x040). Any simulation of the
bootstrap ROM would have dispatched NOP incorrectly and ENTRY_RESET
would have been unreachable.

**Nature:** Behavioral correction. Affects simulation correctness directly.

**Implementation expectation:** This file will be regenerated by the
uasm assembler once the toolchain is complete (Appendix C). The corrected
hand-written version is the bootstrap ground truth until then.

---

## rtl/include/keystone86_pkg.sv

**Why changed:** The file was a severely truncated stub containing only
4 of 20+ entry IDs, 3 of 60+ service IDs, and 3 fault codes. This would
cause immediate RTL compile failures and would mean RTL modules could not
reference the shared namespace at all. Also broke the single-source-of-truth
principle for namespace alignment.

**Nature:** Alignment correction and completion. No architectural changes.

**Implementation expectation:** This file should eventually be generated
by the codegen pipeline from appendix_a_codegen.json. The corrected
hand-written version matches Appendix A exactly and is the ground truth
until the generator is functional.

**What was added:** All 20 entry IDs (with phase markers for phase 2/3
entries), all 60+ service IDs, all 14 fault codes, all 10 commit mask
bits and 11 combined masks, 8 stage field selectors, 16 register names,
16 condition codes, 4 microsequencer state codes.

## tools/spec_codegen/microseq_bootstrap_seed.json

**Why changed:** Seed only declared FETCH_DECODE and EXECUTE states,
omitting WAIT_SERVICE and FAULT_HOLD. The spec (Appendix A, RTL spec,
master design statement) defines four states as mandatory. WAIT_SERVICE
is architecturally required as soon as any SVCW invocation appears, which
happens at Rung 5 (INT_ENTER is a wait-capable service). FAULT_HOLD is
required any time a service returns SR_FAULT.

**Nature:** Alignment correction. Does not change architecture.

**Implementation expectation:** The full microsequencer RTL must implement
all four states per the spec. The corrected seed now serves as the
verification source of truth for this requirement.

---

## microcode/src/shared/sub_fault_handler.uasm

**Why changed:** Bootstrap stub was correct in behavior (ENDI CM_FAULT_END
is the right safe landing) but did not document that the full delivery path
requires INT_ENTER which is not available until Rung 5. A future implementer
looking at this file would not know what to expand it into.

**Nature:** Documentation improvement. No behavioral change to bootstrap stub.

**Implementation expectation:** At Rung 5, this file should be expanded to:
  EXTRACT T4, MF_FC_TO_VECTOR
  SVCW    INT_ENTER
  ENDI    CM_FAULT_END
